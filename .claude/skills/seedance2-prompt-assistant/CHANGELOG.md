### 2. 工作流程优化 - 添加需求确认阶段

**问题**：初始版本在用户提供想法后直接生成提示词，缺少必要信息确认。

**修改**：将工作流程改为两阶段制

#### 第一阶段：需求确认（必做）
- 用户提供想法后，必须先确认以下信息才能生成提示词
- 确认内容：
  1. 视频时长
  2. 影片风格
  3. 情绪基调
  4. 参考素材（图片/视频/音频）
  5. 其他要求

#### 第二阶段：生成专业提示词
- 确认所有信息后，才进入生成阶段

### 3. 提示词输出格式规范化

**问题**：输出格式不够结构化，缺少镜头设计的分秒描述。

**修改**：定义了 6 模块输出格式

| 模块 | 内容 |
|--------|------|
| 角色 | 数量、外观特征、服装、表情状态 |
| 场景 | 地理位置、建筑风格、空间布局、天气光线 |
| 动作/剧情 | 按时间顺序描述的事件和动作 |
| 镜头设计（分秒描述） | 按时间分段，每1-2秒详细描述 |
| 摄影风格 | 画面质感、色彩/黑白调性、光影效果、颗粒感 |
| 音乐建议 | 音乐风格、节奏快慢、情绪配乐、卡点需求 |

### 4. 镜头设计分秒描述指南

**新增内容**：
- 必须按时间分段，每个段落标明时间范围（如 0-2秒、2-4秒）
- 分段原则表格：

| 时长建议 | 分段方式 | 说明 |
|---------|---------|------|
| 1-3秒 | 1-2段 | 每段1-2秒，简洁有力 |
| 4-6秒 | 2-3段 | 起幅-过程-落幅结构 |
| 7-10秒 | 3-5段 | 更细致的镜头节奏 |
| 10秒以上 | 5段以上 | 可设置情绪高潮点 |

### 5. 视频信息确认改为键盘选择格式

**问题**：初始使用"填写方式"，用户体验不够直观。

**修改**：改为数字键盘选择界面

```
═════════════════════════════════════════════════════════════════
                        📹 视频信息确认
═════════════════════════════════════════════════════════════════

请按数字键选择，然后按回车确认：

┌─────────────────────────────────────────────────────────────────────────┐
│ 1. 视频时长                                                       │
│   [1] 1-3秒   [2] 4-6秒   [3] 7-10秒   [4] 10秒以上            │
├─────────────────────────────────────────────────────────────────────────┤
│ 2. 影片风格                                                       │
│   [1] 黑白胶片     [2] 古风武侠     [3] 电影感写实                 │
│   [4] 动漫国漫     [5] 赛博朋克     [6] 清新治愈                 │
└─────────────────────────────────────────────────────────────────────────┘
```

**优势**：
- 用户只需输入数字（如 1-1  2-2  3-3）
- 更符合终端交互习惯
- 选项清晰，一目了然

### 6. 添加参考图片强制分析机制（关键修复）

**问题案例**：
- 用户提供了6宫格分镜图，但AI没有使用图像分析工具仔细查看内容
- AI假设了图片中的场景（如"乡间小径"）与实际不符（实际是石板巷子）
- 用户明确要求后才使用工具分析，导致生成的提示词不准确

**修改**：添加了"处理参考图片的关键原则"章节，强制要求：

#### 新增核心规则
1. **当用户提供参考图片时，必须使用图像分析工具**
   - 不允许凭想象或猜测描述图片内容
   - 对于分镜图/多宫格图片，必须逐格分析

2. **分镜图分析提示词模板**
   ```
   请详细描述这张X宫格分镜图中每个格子的内容，包括：
   - 每个格子里有什么角色
   - 在做什么动作
   - 场景是什么样的（环境、背景、物品等）
   - 镜头角度如何
   请按照从左到右、从上到下的顺序逐个格子描述。
   ```

3. **常见错误警示**
   - ❌ 不要：只看一眼就凭想象描述
   - ❌ 不要：使用"可能""推测"等模糊词汇
   - ❌ 不要：添加图片中没有的元素
   - ✅ 必须：使用图像分析工具获取准确信息
   - ✅ 必须：严格按照图片内容编写提示词

#### 影响范围
- 修改了"与用户交互时的注意事项"，添加"参考图片必须分析"规则
- 新增独立的"处理参考图片的关键原则"章节

**预期效果**：避免因未分析参考图片导致的提示词内容偏差

### 7. 添加交互式选择机制（用户体验优化）

**问题案例**：
- 原来需要用户输入数字编号（如 1-1 2-2），操作不够直观
- 用户需要查看选项清单，然后输入对应数字，容易出错

**修改**：

#### 新增交互式确认方式
- 使用 AskUserQuestion 工具让用户通过点击选项进行选择
- 每次提问都是独立的，用户可以直接点击选择对应的选项
- 操作更直观便捷，无需记忆编号

---

### 8. 添加选项定制化机制（智能推荐）

**问题案例**：
- 原来的选项是固定的，不考虑参考图片和故事内容
- 提供的选项可能与用户需求不匹配（如黑白图片推荐彩色风格）

**修改**：新增"选项定制化规则"章节

#### 核心原则
选项必须与参考图片和故事内容相关联

#### 新增定制化表格

**根据参考图片风格定制**
| 图片类型 | 推荐风格选项 |
|---------|--------------|
| 黑白复古 | 黑白胶片、电影感写实 |
| 色彩鲜艳 | 清新治愈、动漫国漫 |
| 暗色调 | 赛博朋克、冷峻肃杀 |
| 明亮温暖 | 热闹欢快、优雅唯美 |
| 古代场景 | 古风武侠、电影感写实 |

**根据场景类型定制**
| 场景类型 | 推荐情绪选项 | 推荐运镜方式 |
|---------|--------------|--------------|
| 室内静态 | 优雅唯美 | 稳定镜头 |
| 户外奔跑 | 热闹欢快 | 手持晃动 |
| 悬疑场景 | 悬疑神秘 | 慢镜头 |
| 动作场面 | 激烈紧张 | 快镜头 |

**根据角色状态定制**
| 角色状态 | 推荐描述重点 |
|---------|--------------|
| 奔跑中 | 侧重动态姿态、发丝飞扬、脚步节奏 |
| 静止中 | 侧重表情细节、眼神变化、微表情 |
| 互动中 | 侧重角色间的关系、动作呼应、目光交流 |

**根据分镜内容定制**
- 分析每个格子的叙事顺序
- 根据格子数量和内容分布，推荐合适的时长分段
- 根据画面中的镜头角度，参考原始构图设计运镜

**示例**：
- 分镜中有石阶场景 → 推荐"手持晃动"运镜
- 分镜中有特写镜头 → 推荐包含"特写"景别的分段设计
- 分镜中有宽阔场景 → 推荐"推远收尾"的镜头设计
